Our coding + genAI toolset: intro and setup

An introduction to, and setup of, the main tools we’ll use in the workshop

1 Introduction

In the rest of this workshop, we’ll primarily use the following set of tools to run code, access data, and make use of agentic AI tools:

  1. A code editor, VS Code

  2. A shared computing environment, OSC (the Ohio Supercomputer Center)

  3. Generative and Agentic AI tools, GitHub Copilot and Claude Code, in the form of plugins for VS Code

In this session, we will go through the necessary steps be able to use this setup, providing context about the various tools and their use in and after the workshop.

This same setup is what I use in my own work, and you can continue use it after the workshop (with some caveats regarding the AI tools, which we’ll discuss). Getting this setup configured on your own computer is a bit of a process, but it is well worth it in my opinion.

2 Exploring VS Code

2.1 Introduction

VS Code is basically a fancy text editor. To emphasize the additional functionality relative to basic text editors like Notepad, editors like VS Code are also referred to as IDEs: Integrated Development Environments1.

NoteWhy we use VS Code here

It’s a great editor all around, and when working at OSC in particular —at least if you need to run command-line programs, and not just, say, R— it’s by far the most convenient and user-friendly way, in my opinion.

You can also use VS Code at OSC in a web browser via OSC OnDemand, like we did in the pre-workshop session. That’s nice because it requires no installation or other setup.

But in the long run, connecting your own VS Code installation to OSC is preferable. In the context of this workshop, for example: we can’t use AI tools like GitHub Copilot in the OnDemand version of VS Code, but we can when connecting our own VS Code installation to OSC.

You should have already installed VS Code (full name: Visual Studio Code) on your laptop. Go ahead and open it.

2.2 The VS Code User Interface

  • The main part of the VS Code window is the Editor Pane (C above), where you can open and edit text files.

  • The panel at the bottom (D above) contains a terminal, but isn’t open by default.
    In the top menu, click Terminal > New Terminal to open it.

  • By default, the Primary Side Bar (B above) shows a file explorer, where you can see your files and create new ones, and so on.

  • The Activity Bar (narrow side bar, A above) on the far left has:

    • Icons to toggle between options for the Primary Side Bar. For now, pay attention to the bottom one, Extensions, which we’ll use soon.

    • A (cog wheel icon) in the bottom, with two key options:

      • Command Palette, where you can access all VS Code functionality
      • Settings, where you can configure VS Code

Your turn

  • In the cog wheel menu, click Themes and then Color Theme to explore VS Code color themes and optionally change the color theme to your liking.

  • You can resize panes by dragging the borders between them: try to do this with the Primary Side Bar and the Panel with the terminal.

3 Setting up the OSC connection in VS Code

3.1 Configure the connection to OSC

Now, we’ll set up VS Code to connect to OSC via so-called “SSH tunneling”. Once configured, this is a very convenient and powerful way to access OSC. This process moves a VS Code window entirely to OSC: the terminal, the file system, the code editor, and so on2.

The configuration process below is a one-time setup: once done, you can use it to connect to OSC at any time.

Install the Remote Development extension

  1. In the VS Code Activity Bar, click the Extensions icon at the bottom (left screenshot below).

  1. Now, the Primary Sidebar will show the Extensions view, where you can search for and install extensions. In the search box, type “remote development” and the “Remote Development” extension should appear (right screenshot above).

  2. Click the “Install” button to install this extension.

Setup the connection

Next, we’ll use functionality from this extension pack to set up the connection to OSC:

  1. Open the VS Code Command Palette3 and start typing “remote open”.

  2. Click the top option that shows up, “Remote-SSH: Open SSH Configuration File…”, and then again the top option, a file that contains your computer username and ends in /.ssh/config (see screenshots below):

  1. Paste the following (note Mac vs. Windows instructions) into the file that opens, then:
    • In two places, change User <osc-username> to your OSC username (for example, to User jane).
    • Save the file (File > Save or Ctrl/⌘+S).
    • Windows
    • Mac
    Host osc-cardinal
    User <osc-username>
    HostName cardinal.osc.edu

Host osc-cardinal-compute
    User <osc-username>
    ForwardAgent yes
    StrictHostKeyChecking no
    UserKnownHostsFile=/dev/null
    ProxyCommand ssh osc-cardinal "/usr/bin/salloc -A PAS3454 -t 300 /bin/bash -c 'nc $SLURM_NODELIST 22'"
    Host osc-cardinal
    User <osc-username>
    HostName cardinal.osc.edu

Host osc-cardinal-compute
    User <osc-username>
    ForwardAgent yes
    StrictHostKeyChecking no
    UserKnownHostsFile=/dev/null
    ProxyCommand ssh osc-cardinal "/usr/bin/salloc -A PAS3454 -t 300 /bin/bash -c 'nc \$SLURM_NODELIST 22'"
  2. Open the VS Code Settings (cog wheel => Settings) and search for remote.SSH.connectTimeout. Change the value from 15 to 300 (seconds).
NoteWhat we just did

In this SSH configuration file, we defined two remote “hosts” that we can connect to:

  • osc-cardinal, which will connect to a login node of OSC’s Cardinal cluster. However, because the VS Code program will actually run on the login node when you connect it with SSH-tunneling, the below connection to a compute node is preferred.

  • osc-cardinal-compute will connect to a compute node of OSC’s Cardinal cluster. We can’t connect directly to a compute node but have to request a compute job, which is why that connection is more complicated. This connection will last for 5 hours (300 minutes), as it requests a compute job of that duration.

Because compute jobs (for osc-cardinal-compute) may take a while to start, we also increased the “connection timeout” (amount of time VS Code will wait for the connection to be established) to 5 minutes.

(To connect to other OSC clusters, you can add additional entries to this SSH configuration file, simply replacing cardinal by the name of the other cluster.)

WarningFor after the workshop: Changing the account to be charged for compute time (Click to expand)

The -A PAS3454 part of the connection instructs OSC to charge the compute time to the workshop’s OSC Project. While it’s OK to keep using this for a bit after the workshop, if you have access to another OSC project, you should change PAS3454 to that project’s number.

3.2 Connect to OSC

With the configuration all done, you can now (and in the future) connect to OSC like so:

  1. Open the Command Palette and start typing “SSH connect”

  2. Click on SSH: Connect to Host and then select the second of the connections you just set up, osc-cardinal-compute:

  1. A new VS Code window will open. In the little pop-up at the top, enter your OSC password (this won’t be necessary in the future if you do the optional setup below).

  2. The connection may take a little while (typically 5–30 seconds) to establish, because a compute job has to be requested, assigned resources, and started.

    Once connected, a pop-up may appear, asking whether you “trust the authors” of the folder you ended up in: click Trust and Continue.

    If the connection fails or takes more than a minute to establish, ask us for troubleshooting help. As a last resort, try connecting to osc-cardinal (the login node connection) instead.

3.3 Open a folder on OSC

VS Code takes a specific folder as a starting point in all parts of the program: in its file explorer, in the terminal, when saving files, and so on.

By default, this folder is your Home directory. In this workshop, we’ll only be working in a folder specific to it, so let’s open that one now.

Click File > Open Folder and then enter and select /fs/ess/PAS3454.

This will cause the program to reload and the “trust popup” will appear again4.

In the File Explorer, you should now see files … TBA ….

TipFrom now on: Connecting directly to a specific folder at OSC (Click to expand)

The above two-step procedure (connect to OSC, then open a folder) can be done in one even simpler step, after you’ve connected to that folder at least once, as follows.

After you re-open VS Code (or open a new window with File > New Window), the Recent section in the VS Code “Welcome” document will show the folder(s) you previously opened:

Simply clicking on that line will open that folder and in the process connect to OSC!

NoteOptional setup so you won’t have to enter your OSC password again (Click to expand)

If you take the following steps, you will no longer be prompted for your OSC password when you connect to OSC:

  1. Open the terminal in VS Code (in a window that’s on your own computer, not at OSC).

  2. In the terminal (regardless of the type or operating system), execute the following command to generate a public-private SSH key-pair:

    ssh-keygen -t rsa

  3. You’ll get some (odd-looking!) output and will then be asked several questions. In each case, you can simply press Enter to select the default answer; do this until you get your command prompt back.

  4. Excute the following command to transfer the public key to OSC’s Cardinal cluster, but make sure to replace <user> with your OSC username:

    • Windows
    • Mac
    # Replace <user> by your username, e.g. "jelmer@cardinal.osc.edu"
cat ~/.ssh/id_rsa.pub | ssh <user>@cardinal.osc.edu 'mkdir -p .ssh && cat >> .ssh/authorized_keys'
    # Replace <user> by your username, e.g. "jelmer@cardinal.osc.edu"
ssh-copy-id <user>@cardinal.osc.edu

  5. All done! Test if it works by connecting to OSC once again.

4 Setting up GitHub Copilot

4.1 Background

GitHub Copilot is an generative and agentic AI tool that can help you write code and perform other tasks. It’s the first of two similar tools we will use in this workshop, the other being Claude Code.

GitHub Copilot is not the same as the “general” Microsoft Copilot. And while GitHub Copilot is a Microsoft product, hence the similar name, you cannot –currently– use it institutionally at OSU.

Instead, access runs through a personal GitHub account. It’s easy and free to create a GitHub account, and you may have already had an account, or created one following the optional pre-workshop material.

Keep in mind that GitHub is a much more general platform than just GitHub Copilot, widely used for sharing and collaborating on code. Nevertheless, with your GitHub account, you are nowadays also automatically enrolled in the Free Tier of GitHub Copilot.

NoteGitHub Copilot tiers

This free tier will work for the purposes of this workshop, but is not sufficient for sustained GitHub Copilot use.

You may have done the optional pre-workshop setup step to obtain GitHub Pro access via your academic credentials. If that was approved (should be quick) and processed (72 hours later), you’ll now have much more extensive, though not unlimited, access to GitHub Copilot.

If you didn’t do that, there isn’t much point in doing it right now, because you won’t obtain access during the workshop. But feel free to do so in a break or after the workshop.

While there’s also a stand-alone app and web-based interface for GitHub Copilot, we will use it within VS Code, where it is available as a pre-installed plugin. The setup below only serves to log in to your GitHub account or create one along the way.

5 Setting up Claude Code

5.1 Background

Like with GitHub Copilot, the Anthropic product Claude Code comes in various forms, but we will use it within VS Code, where it is available as a plugin.

Unlike with GitHub Copilot, we will not use a personal account to access Claude Code. Instead, we will use an access key that the workshop instructors have obtained using OSU OTDI’s Generative AI Service.

The advantages of this approach —not just during this workshop, but for you moving forward— are:

  1. It is approved at OSU for use with institutional data, all the way to the most sensitive category (S4 classification). 🎉
  2. At least in principle, you can get a work-related rather than personal funding source for your AI usage.
TipWait, why are we showing GitHub Copilot at all, again?

  • For academics, there is a generous free personal option for GitHub Copilot, which is not the case for Claude Code.

  • You may not be able to set up the abovementioned Generative AI Service because you don’t have a work-related funding source to pay for it.

  • Even though it’s ideal to use a tool approved for private data, you may not be working with private data at all, or be able to get help without exposing it. If so, then GitHub Copilot is a perfectly fine option.

  • Seemingly minor but very handy: only GitHub Copilot has inline code suggestions. (And: if you only use the inline code suggestions, you never have to worry about data privacy.)

WarningDon’t get confused about data approval contexts

Claude Code is only approved for use with institutional data when it is accessed via an access key (API key) from OTDI’s Generative AI Service. If you pay for a personal Anthropic account and use that to access Claude Code, you are not using it in an approved way.
