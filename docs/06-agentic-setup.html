1 Introduction

As mentioned in the previous section:

In the rest of this workshop, we’ll primarily use the following set of tools to run code, access data, and make use of agentic AI tools:

  1. A code editor, VS Code
  2. A shared computing environment, OSC (the Ohio Supercomputer Center)
  3. Generative and Agentic AI tools integrated in our code editor, GitHub Copilot and Claude Code

After configuring and testing the VS Code-OSC connection in that section, we will now set up the two agentic AI tools. Don’t despair, this should be a lot more straightforward!

2 Setting up GitHub Copilot

2.1 Background

GitHub Copilot is an generative and agentic AI tool that can help you write code and perform other tasks. It’s the first of two similar tools we will use in this workshop, the other being Claude Code.

GitHub Copilot is not the same as the “general” Microsoft Copilot. And while GitHub Copilot is a Microsoft product, hence the similar name, you cannot –currently– use it institutionally at OSU. Instead, access runs through a personal GitHub account. It’s easy and free to create a GitHub account, and you may already have one from previous usage or because you followed this optional pre-workshop setup step.

Keep in mind that GitHub is a much more general platform than just GitHub Copilot, widely used for sharing and collaborating on code. Nevertheless, with your GitHub account, you are nowadays also automatically enrolled in the GitHub Copilot free plan.

While there’s also a stand-alone app and web-based interface for it, we will use GitHub Copilot within VS Code, where it is available as a pre-installed plugin. Therefore, little setup is needed: the steps below only serve to log in to your GitHub account or create one along the way.

TipPros of GitHub Copilot

  • For academics, there is a generous (more so for teachers than students) free personal option for GitHub Copilot, which is not the case for Claude Code.

  • Even though it’s ideal to use a tool approved for private data, you may not be working with private data at all, or be able to get help without exposing it. If so, then GitHub Copilot is a perfectly fine option.

  • Seemingly minor but very handy: only GitHub Copilot has inline “code completion” features (And: if you only use these, any data privacy concerns can be circumvented.)

NoteGitHub Copilot plans

The GitHub Copilot free plan will work for the purposes of this workshop, but is not sufficient for sustained Copilot use.

You may have done the optional pre-workshop setup step to obtain GitHub Copilot Student/Pro access via your academic credentials. If that was approved (should be quick) and processed (72 hours later), you’ll now have more extensive, though not unlimited, access to GitHub Copilot. For more details, see our GitHub setup page

If you didn’t do that, there isn’t much point in doing it right now, because you won’t obtain access during the workshop given the 72-hour processing time. But feel free to do so in a break or after the workshop.

2.3 One example prompt

Let’s give it one quick try, before we go into more detail in the next sessions. In the Copilot prompt box (which should have the ghost text “Describe what to build”), enter:

How many reads are in data/iqbal/SRR31869590_1.fastq?

Because we’re asking Copilot for information it can only get after actually doing something (running a command), it will by default ask for permission to do so:

Click Allow and it will run the command and return the answer:

What did we just witness?

This example shows how in-editor AI like GitHub Copilot can perform actions themselves, rather than just providing information or code snippets. In technical terms, as part of this agentic behavior, it “calls a tool” (or: “performs tool calls”) that allow it to run arbitrary commands in the terminal.

We’ll soon see another type of extremely useful in-editor agentic behavior, where it can read and/or edit your code. This latter type of behavior is what we’ll use most in this workshop. That’s because broadly speaking, the example above notwithstanding, the goal in this workshop is not to let AI analyze the data for us, but rather to let it help us write code to appropriately analyze the data with existing non-AI programs.

3 Setting up Claude Code

3.1 Background

Like with GitHub Copilot, the Anthropic product Claude Code comes in various forms, but we will use it within VS Code, where it is available as a plugin.

Unlike with GitHub Copilot, we will not use a personal account to access Claude Code. Instead, we will use an access key that the workshop instructors have obtained using OSU OTDI’s Generative AI Service. Moving forward, we will refer to this type of access key as the “OSU LiteLLM API key”1.

The advantages of using Claude Code via the OSU LiteLLM API key —not just during this workshop, but for you moving forward— are:

  1. It is approved at OSU for use with any type of institutional data (S2–S4). 🎉
  2. OTDI currently has an initiative where anyone at OSU can get $100 worth of AI credits via their own OSU LiteLLM API key (see the bottom of this page). 🥳
  3. At least in principle, you can get a work-related rather than personal funding source for additional (as in, beyond this workshop and beyond your free key) AI usage.
WarningDon’t get confused about data approval contexts (Click to expand)

Claude Code is only approved for use with institutional data when it is accessed via an access key (API key) from OTDI’s Generative AI Service. If you pay for a personal Anthropic account and use that to access Claude Code, you are not using it in an approved way.

3.2 The setup steps

  1. In the Extensions view, search for “claude” and install the extension “Claude Code for VS Code”:

  1. To authenticate to Claude Code, we will use a settings file where we tell Claude about our API key and settings associated with it. In the terminal, copy this file to your Home directory (where it will be automatically read by Claude Code):

    cp /fs/scratch/PAS3454/share/settings.json ~/.claude/settings.json
    NoteWhat does this file contain? (Click to expand)

    We can’t share this file on the website in full, because it contains an API key that cannot be exposed publicly. But for your reference, the contents of this file minus the API key (here replaced by REPLACE_WITH_API_KEY) are as follows:

    {
"theme": "auto",
"env": {                                                                                                                                                                                        
    "ANTHROPIC_BASE_URL": "https://litellm.cloud.osu.edu",                                                                                                                                        
    "ANTHROPIC_AUTH_TOKEN": <REPLACE_WITH_API_KEY>,
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "claude-opus-4-6",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "claude-sonnet-4-6",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "claude-haiku-4-5-20251001",
    "CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS": "1"                                                                                                                                        
    }   
}

  2. A second tab should now be available in the side bar with Copilot, named “CLAUDE” — click on that (if you don’t see it, you may have to widen the side bar!):

    NoteAlternative way of opening a Claude Code window (Click to expand)

    Once installed, a Claude Code icon will appear in the topright of the editor pane:

    Clicking this icon will also open a Claude Code window, but this one will be opened within the editor pane rather than in the sidebar.

  4. Simply wait, and if all goes well, Claude will process the settings file behind the scenes, and switch to an interface similar to the Copilot one:
